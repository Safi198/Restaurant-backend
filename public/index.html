Restaurant Backend API Documentation

Explore Our API

Manage various aspects of a restaurant including products, user authentication, orders, and cart management.

Available Endpoints

  • GET: /api/products - Fetch All Products

    Retrieve a complete list of products available in the restaurant. This endpoint helps you get an overview of all items currently offered.

    GET /api/products HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app

  • GET: /api/products/:id - Fetch a Product by ID

    Retrieve detailed information about a specific product by its ID, including pricing, ingredients, and availability.

    GET /api/products/12345 HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app

  • POST: /api/auth/register - Register a New User

    Create a new user account by providing a unique email and password. This allows users to access personalized services and manage their orders.

    POST /api/auth/register HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Content-Type: application/json

    {
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123"
    }

  • POST: /api/auth/login - User Login

    Authenticate a user by providing an email and password. Returns a JWT token upon successful login.

    POST /api/auth/login HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Content-Type: application/json

    {
    "email": "john@example.com",
    "password": "password123"
    }

  • GET: /api/cart - Get the User's Cart

    Retrieve the current user's cart, including all products and quantities.

    GET /api/cart HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Authorization: Bearer YOUR_JWT_TOKEN

  • POST: /api/cart/add - Add a Product to the Cart

    Add a product to the user's cart by specifying the product ID and quantity.

    POST /api/cart/add HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Content-Type: application/json
    Authorization: Bearer YOUR_JWT_TOKEN

    {
    "productId": "12345",
    "quantity": 2
    }

  • POST: /api/cart/remove - Remove a Product from the Cart

    Remove a specific product from the user's cart by providing the product ID.

    POST /api/cart/remove HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Content-Type: application/json
    Authorization: Bearer YOUR_JWT_TOKEN

    {
    "productId": "12345"
    }

  • GET: /api/orders - Get All Orders for the User

    Retrieve a list of all orders placed by the current user.

    GET /api/orders HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Authorization: Bearer YOUR_JWT_TOKEN

  • POST: /api/orders - Create a New Order

    Create a new order using the current contents of the user's cart.

    POST /api/orders HTTP/1.1
    Host: restaurant-backend-production-5748.up.railway.app
    Content-Type: application/json
    Authorization: Bearer YOUR_JWT_TOKEN

    {
    "cartId": "67890",
    "address": "123 Main St, City, Country"
    }
